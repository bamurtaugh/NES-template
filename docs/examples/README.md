# Example Categories

This directory contains detailed documentation for each category of NES examples in the repository.

## Category Overview

| Category | Files | Description | Difficulty |
|----------|-------|-------------|------------|
| [1-Changing_Intent](./1-changing-intent.md) | `stats.cpp`, `geometry.ts` | Modify existing code to change its purpose | ⭐ Beginner |
| [2-Adding_Logic](./2-adding-logic.md) | `extension-sample/`, `game.py` | Add new functionality to existing code | ⭐⭐ Intermediate |
| [3-Changing_Fixing_Logic](./3-changing-fixing-logic.md) | `js-game/`, `scenarios.ts` | Fix bugs and improve existing logic | ⭐⭐ Intermediate |
| [4-Refactoring](./4-refactoring.md) | `java-sample/` | Restructure and improve code organization | ⭐⭐⭐ Advanced |
| [5-Fuller](./5-fuller.md) | `task-manager-app/` | Complete application with multiple NES scenarios | ⭐⭐⭐ Advanced |

## Learning Path

### For Beginners
1. Start with **1-Changing_Intent** to understand basic NES concepts
2. Practice with simple modifications to see how NES responds
3. Observe patterns in the suggestions

### For Intermediate Users
1. Explore **2-Adding_Logic** to learn about feature addition
2. Work through **3-Changing_Fixing_Logic** for debugging scenarios
3. Compare suggestions across different languages

### For Advanced Users
1. Dive into **4-Refactoring** for complex restructuring scenarios
2. Study **5-Fuller** for real-world application patterns
3. Experiment with custom scenarios based on these templates

## Quick Start Guide

Each category includes:
- **Overview**: What the example demonstrates
- **Prerequisites**: What you need to know
- **Step-by-Step Instructions**: Detailed walkthrough
- **Expected Results**: What NES should suggest
- **Variations**: Alternative approaches to try
- **Troubleshooting**: Common issues and solutions

## Using the Examples

1. **Read the Category Guide**: Start with the overview for your chosen category
2. **Set Up Your Environment**: Ensure Copilot is active and configured
3. **Follow the Instructions**: Make the suggested edits step by step
4. **Observe and Learn**: Pay attention to how NES responds
5. **Experiment**: Try variations to deepen your understanding
